# Literary Studio — инструкция для писателя

Пошаговый рабочий процесс: что делает автор и что проверяет на каждом этапе. Ссылки — на разделы `literary-studio-tz.md`.

---

## Этап I. Планирование

### 1. Создать/выбрать книгу
- Новая серия — создаётся с нуля; новая книга в существующей серии — Библия мира и Авторский стиль (§3.5–3.6) уже есть и наследуются автоматически
- Заполнить реквизиты: заголовок, подзаголовок, жанр, теги, краткая и полная аннотация

**Проверить:** для книги 2+ в серии — открыть Библию мира и свериться, что новый замысел не противоречит уже установленному канону.

### 2. Набросать Идеи
Свободные заметки «а что если» — черновик-накопитель, не канон (§1). Ничего проверять не нужно, это черновая стадия.

### 3. Построить Матрицу романа (§3.1)
- Определить сюжетные линии (столбцы) — не обязательно персонажи, могут быть организации, локации, мета-линия
- Расписать биты по строкам

**Проверить:** каждая линия имеет разумную частоту появления (не пропадает на полкниги без причины); мета-линия «общий поворот» логично собирает остальные линии в каждом акте.

### 4. Расставить события в Таймлайне (§3.2)
Перенести ключевые биты матрицы на временную шкалу мира — даты, возраст персонажей, интервалы между событиями.

**Проверить:** нет ли нестыковок по времени между сценами, которые кажутся соседними в матрице, но разнесены во времени внутри мира.

### 5. Построить Карту связей (§3.3)
Добавить персонажей/организации/локации как узлы, прописать связи между ними.

**Проверить:** совпадает ли граф связей с тем, что уже заложено в матрице (например, если два персонажа взаимодействуют в бите матрицы, у них должна быть связь в графе).

### 6. Завести карточки персонажей и арки (§3.4)
Для ключевых персонажей — базовые поля + варианты имени с правилом контекста + таблица арки (цель/конфликт/что узнал по ключевым сценам).

**Проверить:** у каждого значимого персонажа арка имеет все четыре типа бита (завязка/испытание/перелом/развязка), иначе дуга скорее всего оборвана.

### 7. Пополнить Библию мира (§3.5)
Завести карточки Локаций, Организаций, Правил, Термина по мере необходимости — с полем «Источник/Первое упоминание» (сцена, где читатель впервые об этом узнает).

**Проверить:** область действия карточки (серия/локальная) — не плодить в общей библии одноразовых персонажей одной книги.

### 8. Свериться с Авторским стилем (§3.6)
Для новой серии — задать Стоп-лист, пунктуационные правила, POV/время, тональность. Для продолжения — просто иметь в виду, что уже действует.

## Этап II. Написание

### 1. Открыть сцену
Из дерева навигации или прямо из ячейки Матрицы романа (клик по пустой ячейке создаёт и связывает сцену).

### 2. Писать текст
- Использовать AI-помощника **Со-автор** для черновика/продолжения сложных мест
- По ходу — **Читатель** для быстрой проверки понятности абзаца, если сомневаетесь

**Проверить сразу по ходу:** если в сцене впервые появляется термин или правило — завести для него карточку в Библии мира с этой сценой как «Источник» (§3.5), не откладывать на потом.

### 3. Отмечать статус сцены (§4.2)
`Черновик` — пока пишете. Не переводите в следующий статус, пока сцена не закончена целиком — статус нужен для честной картины прогресса по книге.

### 4. Следить за прогрессом (§4.3)
Целевой объём в знаках на сцену/главу/книгу, прогресс-бар, счётчик за сегодня.

### 5. Периодически смотреть график темпа/плотности (§4.4)
Раз в несколько глав — не после каждой сцены, чтобы не терять цельность взгляда на книгу.

**Проверить:** нет ли длинных провисаний темпа (плоская линия напряжения на протяжении нескольких сцен подряд) — это сигнал для этапа III, не повод останавливать написание сейчас.

### 6. Матрица обновляется автоматически
Ячейка со связанной сценой отмечается как написанная — отдельно ничего делать не нужно, только проверять визуально прогресс по линиям.

## Этап III. Редакторский пайплайн

Начинается, когда черновик главы или книги целиком закончен (все сцены в статусе «Черновик» дописаны).

### 1. Критик
Запустить структурный анализ: дыры в сюжете, логические нестыковки, сверка с Библией мира.

**Проверить:** каждое найденное замечание — либо исправить, либо осознанно отклонить (не пропускать молча).

### 2. Редактор
Построчная правка: грамматика, стиль, проверка по Стоп-листу и правилам пунктуации из Авторского стиля, проверка согласованности имён персонажей по правилам из карточки (§3.4).

**Проверить:** отчёт о срабатываниях Стоп-листа — не механически удалять все совпадения, некоторые уместны в контексте.

### 3. Читатель
Финальная проверка «первого взгляда» — понятность, эмоциональный отклик, слепые пятна, которые автор перестал замечать.

### 4. Правки и повторные раунды
Внести правки по замечаниям всех трёх помощников. При существенных изменениях — повторить раунд Критик → Редактор → Читатель.

**Проверить между раундами:** открыть diff между версией до и после раунда (§6) — убедиться, что правки не создали новых противоречий с Библией мира или таймлайном.

### 5. Смена статуса
`На редактуре` → `Вычитано` → `Финал`, по мере прохождения раундов.

## Этап IV. Экспорт и публикация

### 1. Финальная сборка
Экспорт в FB2/DOCX/EPUB (§7) — приоритетные форматы для площадок. PDF — только для личного превью/печати, не для загрузки на платформы.

### 2. Продакшн-ассеты (§8)
Подготовить/обновить: обложка (с версионированием), промпты для трейлера, TTS-файлы, файлы под мерч.

### 3. Заполнить раздел «Публикация» (§9)
Платформа, статус, SEO-поля площадки, заметки по ограничениям площадки.

**Проверить:** формат загрузки соответствует требованиям площадки (например, не PDF, если площадка требует fb2/docx/epub для генерации аудиокниги — как в опыте с Litres).

### 4. Обновить статус публикации по факту
`Отправлено` → `Опубликовано` (или `Снято с публикации`, если понадобится доработка).

---

## Общий принцип цикла
Для книги 2+ в серии цикл начинается заново с Этапа I, но Библия мира и Авторский стиль уже не создаются с нуля — только пополняются и сверяются. Именно поэтому оба вынесены на уровень серии, а не книги (§1).
